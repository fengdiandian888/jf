> **适用项目：** E:/减脂计划/ （GitHub Pages · 4个单文件HTML）  
> **适用人员：** 单人维护（AI辅助开发）  
> **最后更新：** 2026-06-24

## 一、审查频率

| 场景 | 审查级别 | 触发条件 |
|------|---------|---------|
| 日常提交 | **轻量审查** | 改字体/调间距/换颜色等纯CSS变更 |
| 内容更新 | **标准审查** | 改训练动作/饮食数据/作息时间/BV号 |
| 架构变更 | **深度审查** | 新增页面/重构JS/改变数据结构 |

## 二、审查清单

### 2.1 轻量审查（5分钟）

每次 `git push` 前必查：

- [ ] **CSS变量完整性** — 新增的颜色值是否用了 `var(--xxx)`？是否存在的变量？
- [ ] **排版层级** — 改动后的字号属于 L1-L6 哪一级？（见下方层级表）
- [ ] **字体定义** — `font-family` 是否用 `var(--font)`？
- [ ] **Tab栏/Bottom Nav** — 四个文件的激活态 `class="active"` 是否正确？
- [ ] **无遗留调试代码** — 检查 `console.log`、`alert`、`<!-- TODO -->`

### 2.2 标准审查（15分钟）

改动共享数据时必须逐项核对：

#### 饮食数据
- [ ] 蛋白质目标：**100g/天**（早27 + 午~20 + 晚53）
- [ ] 晚餐蛋白来源：鸡胸240g / 瘦牛肉240g → 蛋白53g
- [ ] 晚餐蔬菜：**300g**（黄瓜/丝瓜/菠菜/生菜/苋菜轮换）
- [ ] 晚餐碳水：**半根玉米80g / 南瓜200g / 红薯100g**
- [ ] 饮水：**5次×400ml** = 2.0L + 早晚各200ml = 2.2-2.8L
- [ ] 饥饿三级预案：水→黄瓜→牛奶

#### 训练动作
- [ ] **拉日**：弹力带下拉 / 哑铃划船 / 弹力带面拉 / 哑铃弯举
- [ ] **推日**：窄距俯卧撑 / 前平举（替代推举）/ 臂屈伸 / 肩外旋
- [ ] **腿日**：自重深蹲 / 臀桥 / 卷腹 / 平板支撑
- [ ] **禁用动作**：肩上推举 / 宽距俯卧撑 / 哑铃飞鸟 → 在推日相关章节明确标注
- [ ] **周六轻量维持**：明确标注"不求力竭"和"永远不升级"
- [ ] **肩康复**：BV1WM411U7yq · 3'×2~3遍 · 每天必做

#### 作息时间
- [ ] 起床：**07:00** | 早上运动：**07:00-07:30**
- [ ] 早餐：**09:00** | 午餐：**12:00** | 晚餐：**19:00**
- [ ] 晚上训练：**21:00-22:00** | 关灯：**00:00**
- [ ] 通勤方式：**小电动**（非单车）
- [ ] 午饭后散步 10-15'

#### 健康警示
- [ ] 右肩隐痛描述正确 + 禁用/替代方案完整
- [ ] 重度打鼾 OSA 说明 + 侧卧 + 减重治疗
- [ ] BMI 29.4 → 目标 BMI < 22（对应63-70kg）

### 2.3 深度审查（30分钟）

涉及 JS 逻辑改动时额外核查：

- [ ] **天气逻辑** — 两个文件（今日计划/全面计划）是否用了同一套缓存策略？是否都有降级？
- [ ] **数据源一致性** — 共享数据（蛋白/BV号/动作名）是否在多个文件里各写一份？如果是，建议提取。
- [ ] **事件绑定** — 是否有 `onclick="..."` 内联绑定？建议改为 `addEventListener` 或 `data-*` 属性。
- [ ] **错误处理** — fetch 调用是否有 `.catch()`？Service Worker 注册失败是否至少 `console.warn`？
- [ ] **let vs const** — 不重新赋值的变量必须用 `const`，避免 `var`
- [ ] **PWA 缓存** — 改完 `serviceWorker.js` 是否确认了缓存策略（Network First）？

## 三、排版层级标准（每次 CSS 改动用）

| 层级 | 字号 | 字重 | 颜色 | 适用场景 |
|------|------|------|------|---------|
| **L1** | 28-32px | 800 | `var(--text)` | 页面主标题、今日日期 |
| **L2** | 20-22px | 700 | `var(--text)` | 卡片标题（如"基础数据"） |
| **L3** | 24-28px | 800 | 语义色 | 关键数据（100g/2.5L/90′/7h） |
| **L4** | 14-15px | 500-600 | `var(--text)` | 正文、描述、动作名 |
| **L5** | 11-12px | 600-700 | `var(--sub)` | 标签、分类、时间戳 |
| **L6** | 12-13px | 400 | `var(--sub)` | 辅助信息、备注 |

**规则：**
- L1 与 L4 之间必须有 ≥2 倍的字号差
- L3（数据）必须用语义色（蛋白=绿、饮水=蓝、运动=橙、睡眠=紫）
- L5 标签必须加 `letter-spacing: 0.04em`
- 不允许任何文字小于 11px
- 不允许 `text-shadow` 效果

## 五、审查流程（What to do 而非 How to feel）

### 审查步骤（每次改代码后执行）

```
1. git diff --stat → 确认改了几个文件
2. 对照"审查清单"逐项打勾（选择对应级别）
3. 打开手机浏览器预览 → 抽查 2-3 个关键数据点
4. git commit → 提交信息包含版本号（vXX）和改动摘要
5. git push → 确认 GitHub Actions 或 Pages 更新
```

### Commit 信息格式

```
vXX: 简短描述（改动范围）

示例：
v19.2: 修正三大加速杠杆为v10最新版
v19.1: 全部折叠内容默认展开
v19: 真正排版层级 — 参考健身App截图 + 副色#9ca3af
```

## 六、已知技术债务（待重构清单）

以下问题已识别但暂不阻塞当前迭代：

1. **CSS 变量名不统一** — `--warn` vs `--warning` vs `--red` 在四个文件中混用
2. **组件重复** — Tab栏/Bottom Nav 在 4 个文件中各写一套
3. **共享数据分散** — BV号/蛋白克数/动作名散落在多处
4. **天气逻辑两份** — 今日计划和全面计划各自独立实现
5. **字体定义方式不一致** — 有的用 `var(--font)`，有的直接写
6. **缺少 HTML meta description** — 影响社交媒体分享预览
7. **无障碍访问缺失** — 无 ARIA / 语义标签 / 色差验证
